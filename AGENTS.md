# Trend Utility Factory Agent Rules

## Token-Saving Rules
- Never regenerate the entire factory for one project.
- Never rewrite existing templates unless explicitly required.
- Prefer JSON brief changes over code changes.
- Prefer scripts over manual file creation.
- Prefer small diffs.
- Do not inspect unrelated files.
- Do not refactor unless required.
- Do not add dependencies without approval unless absolutely necessary.
- Use existing components and SEO helpers.
- If a project can be generated by CLI, run the CLI instead of manually coding it.
- If build fails, fix only the failing file.
- Codex should only be used for build failures, broken templates, TypeScript errors, or final minimal polish.

## Windows Compatibility Rules
- Always use `npm.cmd` instead of `npm`.
- Always use `npx.cmd` instead of `npx`.
- Do not rely on Bash.
- Use Node.js `fs` and `path` scripts for copying, deleting, writing, and reading files.
- Generated scripts must work on Windows.

## Safety Rules
- Work only inside this repository.
- Do not read or modify files outside this repository.
- Do not read `.env` files, SSH keys, browser profiles, or personal folders.
- Do not deploy, spend money, create external accounts, or call paid APIs unless explicitly asked.
- Do not install global packages without asking.
- Do not change system settings.

## Product Rules
- No games.
- No POS focus.
- No World Cup focus.
- Do not base ideas on personal history.
- Build useful trend utilities for general internet demand.
- Avoid keyword stuffing.
- Avoid copying brands or logos.
- Add a disclaimer for finance, health, legal, official, or sensitive topics.
- Every project should give users a real output: calculation, checklist, plan, quiz result, tracker result, generator output, simulation, share card, or printable result.

## Codex Minimal-Fix Rules
- Start with the failing command and the smallest relevant file set.
- Fix only the failing file unless the error proves a shared helper is wrong.
- Rerun the exact failing command after the fix.
- Record recurring build failures in `datasets/build-errors`.
- Record useful fixes in `datasets/codex-fixes`.

## Local Robot Workflow
- Use `npm.cmd run agent:research` before asking Codex to invent ideas.
- Use `npm.cmd run agent:daily -- --dry-run --count 4` to preview a daily batch.
- Use `npm.cmd run agent:daily -- --count 4 --force` to let local robots write briefs and generate projects.
- Use `npm.cmd run agent:review -- <slug>` before asking ChatGPT, Gemini, or Codex for final review.
- Use ChatGPT and Gemini as idea/review councils by pasting agent outputs into the documented prompts; do not require paid APIs by default.
- Use Ollama only when it is already installed and intentionally selected with `--provider ollama`.
